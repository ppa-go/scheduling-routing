Documentation des fonctions

solution find_solution ( instance  data,
unsigned int  nb_mutations 
)

Fonction qui retourne une solution pour une instance donnée, en réalisant une descente locale.

Paramètres
dataUne instance
nb_mutationsLe nombre de mutations à réaliser à chaque étape de l'algorithme
Renvoie
Une solution pour l'instance data
Note
L'algorithme utilisé pour construire la solution retournée est le suivant :
  1. Recherche d'une solution "mère" initiale (heuristique)
  2. Faire :
    • Générer nb_mutations solutions "filles", à partir de la solution "mère" (mutations aléatoires)
    • Rechercher la meilleure solution parmi les solutions "fille"
    • Si la meilleure solution "fille" est elle-même meilleure que la solution "mère" actuelle, alors elle devient la prochaine solution "mère".
    Tant que l'une des solutions "fille" est meilleure que la solution "mère".
La solution initiale est trouvée gràce à la fonction find_first_group().
Les mutations sont effectuées sur la répartition des jobs en groupe, avec la fonction mutate().
L'algorithme génère des solutions "filles" jusqu'à ce qu'elles soient deux à deux distinctes.
